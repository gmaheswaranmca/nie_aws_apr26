# AWS Identity and Access Management - Practical Demo

This lab demonstrates **IAM User creation, policy attachment, and IAM login**.

---

# AWS IAM Practical Lab – Create User and Attach Policy

## Objective

Learn how to:

1. Create an **IAM User**
2. Attach **policies**
3. Login using the **IAM User**

---

# Step 1 – Login to AWS Console

1. Open
   [https://console.aws.amazon.com](https://console.aws.amazon.com)

2. Login using your **root account**.

---

# Step 2 – Open IAM Service

1. In the **Search Bar**, type

```
IAM
```

2. Click **IAM – Identity and Access Management**

IAM is used to **manage users, permissions, and access to AWS resources**.

---

# Step 3 – Create IAM User

1. In left menu click

```
Users
```

2. Click

```
Create user
```

3. Enter user name

```
traineruser
```

4. Enable access type

✔ **Provide user access to AWS Management Console**

5. Set password

Select

```
Autogenerated password
```

or

```
Custom password
```

Example

```
Trainer@123
```

6. Click

```
Next
```

---

# Step 4 – Attach Policies

Choose

```
Attach policies directly
```

Search policies and attach:

### Policy 1

```
AmazonS3FullAccess
```

This allows:

* Create bucket
* Upload objects
* Delete objects
* Full S3 control

### Policy 2

```
ReadOnlyAccess
```

This allows:

* Read AWS resources
* Cannot modify resources

Select both policies.

Click

```
Next
```

---

# Step 5 – Review and Create User

Check:

```
User name : traineruser
Policies  :
   AmazonS3FullAccess
   ReadOnlyAccess
```

Click

```
Create user
```

User is created.

---

# Step 6 – Get IAM Login URL

After creation you will see:

```
Console sign-in URL
```

Example:

```
https://123456789012.signin.aws.amazon.com/console
```

Copy this URL.

This is used for **IAM user login**.

---

# Step 7 – Login Using IAM User

1. Open the **IAM login URL**.

You will see:

```
Account ID / Alias
Username
Password
```

Enter

```
Username : traineruser
Password : Trainer@123
```

Click

```
Sign in
```

---

# Step 8 – Verify Permissions

After login try:

1. Open **S3**
2. Create a bucket

You should be able to **create buckets because of S3FullAccess**.

Now try:

1. Open **EC2**

You can **view instances** but cannot modify them due to **ReadOnlyAccess**.

---

# Expected Result

You successfully:

* Created IAM User
* Attached policies
* Logged in using IAM credentials
* Verified permissions

# Notes
```
Users : An IAM user is an identity 
with long-term credentials 
that is used to interact with AWS in an account.

Policies: A policy is an object in AWS 
that defines permissions.

Roles: An IAM role is an identity 
you can create 
that has specific permissions 
with credentials 
that are valid for short durations. 
Roles can be assumed by entities that you trust.

User groups: 
A user group is a collection of IAM users. Use groups to specify permissions for a collection of users.
```
